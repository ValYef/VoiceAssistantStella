# Voice Assistant Stella 
— це програмна система, розроблена в межах кваліфікаційної роботи, яка поєднує автоматичне розпізнавання мовлення (ASR), семантичний аналіз тексту (NLU) та клієнтський застосунок на WinUI 3 для забезпечення голосової взаємодії користувача з інформаційною системою у сфері громадської діяльності.

Система дозволяє користувачу ставити голосові запитання, отримувати текстові відповіді та працює в режимі реального часу завдяки потоковій передачі аудіо через gRPC та швидкій обробці тексту на Python‑сервері.

### Система побудована за клієнт‑серверною та мікросервісною архітектурою і складається з трьох основних компонентів:

**1. Клієнтський застосунок (WinUI 3, C#, .NET 8)**
- Запис аудіо через NAudio.
- Потокова передача аудіо на сервер через gRPC.
- Відображення результатів розпізнавання та класифікації.
- Анімована сфера як індикатор голосової активності.

**2. ASR‑сервіс (Python + Vosk)**
- Приймає аудіо‑потік через gRPC.
- Виконує розпізнавання мовлення.
- Передає текст до NLU‑модуля через FastAPI.

**3. NLU‑сервіс (Python + FastAPI)**
- Попередня обробка тексту.
- Класифікація наміру за TF‑IDF або BERT.
- Обчислення рівня впевненості.
- Повернення відповіді з бази знань.
- Кешування BERT‑ембеддингів для прискорення роботи.

### 🚀 Основні можливості
- Розпізнавання українського мовлення в реальному часі (Vosk + model‑uk‑v3‑lgraph).
- Класифікація намірів користувача за допомогою двох підходів:
    - **TF‑IDF + Logistic Regression / SVM (швидкий статистичний метод)**,
    - **BERT + cosine similarity (контекстний метод підвищеної точності)**.
- Повернення відповідей із внутрішньої бази знань (терміни громадської діяльності, інклюзії, прав людини).
- Потокова передача аудіо через **gRPC**.
- Взаємодія між мікросервісами через **FastAPI**.
- Клієнтський застосунок на **WinUI 3 (.NET 8)** з анімованим індикатором голосової активності.
- Відображення проміжних та фінальних результатів розпізнавання.

### 🧠 Методи обробки тексту
Система підтримує два режими роботи:

1. **TF‑IDF**
- Швидкий, легкий, не потребує GPU.
- Підходить для коротких команд.
- Використовує класичні ML‑алгоритми.

2. **BERT**
- Враховує контекст і семантику.
- Дає вищу точність.
- Потребує більше ресурсів.

### 📊 Тестування
Для оцінювання моделей використовується скрипт: `python evaluate.py`
Він автоматично:
- тренує TF‑IDF та BERT,
- класифікує тестові приклади,
- обчислює Accuracy, F1, Confidence, Latency,
- будує графіки та матриці плутанини.

### 📌 Призначення системи
Voice Assistant Stella створена для:
- пояснення термінів громадської діяльності,
- підвищення цифрової інклюзії,
- спрощення доступу до інформації,
- демонстрації можливостей ML‑моделей у голосових інтерфейсах.
